## Unreleased
- E-commerce (`global` + `pro`): le mail client de confirmation de commande quitte l'ancien envoi Brevo direct du socle e-commerce et passe maintenant par le flux transactionnel AI Studio avec le template `ALL_ALL_INVOICE_MONTHLY`; le périmètre métier reste inchangé, avec envoi seulement sur la première facture éligible et destinataire porté par `CONTACT_EMAIL` côté webhook centralisé.
- Navigation EC (`pro`): sur desktop, la navigation gauche de l'espace pro prend maintenant un peu moins de largeur et libere davantage d'espace pour le contenu des pages; les elements du panneau (logo, liens, CTA) partagent une meme largeur utile pour stabiliser leur centrage, la liste `ul.navbar-nav[data-simplebar]` neutralise ses marges negatives heritees pour eviter un debord horizontal, le bloc `navbar-collapse` est recale proprement dans son conteneur, le shell retire son padding lateral propre pour caler la navigation au plus juste dans le panneau, et le footer bas conserve ses 3 icones en `space-between` avec `svg` non compressibles pour eviter la coupe du pictogramme `Contact`.
- Navigation EC (`pro`): sur mobile, le menu lateral s'ouvre maintenant dans un drawer plus etroit (`min(82vw, 17rem)`) y compris une fois la classe `sidebar-menu` active, avec un override telephone sous `576px` (`min(74vw, 15rem)`); le panneau est maintenant borne en hauteur plutot qu'etire jusqu'en bas de page, avec scroll global du drawer pour garder le footer d'icones dans le flux visible.
- Branding visuel perso (`games` + `global` + `pro`): le jeu envoie maintenant le `File` original au save branding au lieu d'une preview deja compressee, la persistance locale ne reprend plus la main sur une vraie URL serveur au boot, le recadrage final respecte la qualite demandee, la cible visuel `games` monte a `1600x640` avec borne adaptative a la source, et l'EC TdR aligne aussi son upload `visuel` sur cette cible haute.
- Branding visuel perso (`games` + `global` + `pro`): un upload `logo` / `visuel` trop lourd ou en erreur n'echoue plus en silence; `games` reaffiche maintenant le message serveur exact, et l'EC branding affiche une alerte claire au retour.
- Branding (`games` + `global`): réinitialiser le branding depuis une session affilié ne peut plus supprimer par erreur un design réseau TdR hérité; le reset est maintenant strictement borné à la couche session.
- Design réseau (`pro`): la modale de confirmation de sauvegarde cible maintenant le bon formulaire; un conflit d'id `frm` avec le switch multi-compte du shell EC pouvait auparavant envoyer vers la home sans enregistrer le design.
- BO clients (`www`): le lien EC temporaire généré sur la fiche client BO se présente maintenant comme une action de copie, avec feedback `Lien copié`, au lieu d'une navigation directe vers l'URL.
- Auth EC (`pro`): le point d'entree d'authentification expire maintenant aussi les cookies BO `CQ_admin_gate_*` des qu'ils sont consommes; en navigation classique, un ancien passage par le BO ne recolle donc plus automatiquement le dernier compte visite.
- Auth EC (`pro`): la déconnexion EC nettoie maintenant toute la session d'authentification et expire aussi les cookies de délégation éventuels; après un accès par lien temporaire, revenir sur `signin` ne laisse donc plus d'état résiduel bloquant une autre connexion.
- Auth EC (`pro`): l'accès direct historique depuis le BO via cookies admin fonctionne de nouveau; le script d'authentification ne laisse plus les paramètres de routing `GET` écraser une session BO déjà initialisée, ce qui évite le retour parasite vers `signin`.
