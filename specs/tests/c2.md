# Tests manuels — C2 (Bingo WS `case_click` → Canvas API `case_click`)

## Pré-requis
- WS Bingo démarré avec `CANVAS_API_URL` pointant vers `.../global_ajax.php?t=jeux&m=canvas`
- Token service en env (`CANVAS_SERVICE_TOKEN`) + Canvas configuré avec le même secret
- Une session de dev valide (sessionId/token) + un player/grille validés

## 1) Chemin nominal (API OK)
- Cliquer une case depuis un player.
- Vérifier dans les logs WS la présence de `msg="case_click via API ok"` avec `event_id` et `latencyMs`.
- Vérifier côté DB (Canvas):
  - `game_events` contient une ligne `game=bingo`, `action=case_click`, `event_id=<uuid>`
  - `jeux_bingo_musical_grids_clients.box_<N>_timestamp` est bien mis à jour pour la grille.

## 2) Erreur API (ne pas crasher le WS)
- Mettre temporairement `CANVAS_API_URL` sur une URL invalide puis cliquer une case.
- Vérifier dans les logs WS la présence de `msg="case_click via API failed"` (et le WS continue de tourner).

## Où regarder (WS)
- Fichier: `bingo-musical/src/ws/bingo_server.js` (handler `case "case_click"`).
- Logs attendus: tag `LOG_TAGS.CaseClick` et message `case_click via API ok/failed`.
